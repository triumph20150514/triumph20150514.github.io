第一个博客，一个新开始

搭建它老费劲了，折腾了几遍才弄好。记录下过程

  • 所有需要的安装包 Node.js git githup账号，若是以Github为站点的话需要，因为个人以前已经配置好了GitHub这里也就不写啦
  • Github创建一个仓库，名字为GitHub的用户名。看的很多博客这样要求的，有点不知索然
  • 然后就是使用Cmd 查看是否安装好了

  • 然后就是下载所需要的文件也就是Hexo

  • 下载Hexo 通过 命令：$ npm install -g hexo-cli

  • git –version node -v 查看是否安装成功
  • 然后使用npm命令下载Hexo文件，首先你需要创建一个文件夹来存放它
  • 然后使用 git bach 来运行npm命令，它会下载所需要的文件
  • Hexo官网
  • $ hexo init
  • $ cd
  • $ npm install
  • 这样所需要下载的东西已经准备好了
  • 新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。
  • $ hexo new [layout] 例如： hexo new “ My Title “ //layout 为布局格式可以不指定，它为默认格式
  • $ hexo generate 生成静态文件

选项 描述

  • hexo d, –deploy 文件生成后立即部署网站
  • hexo w, –watch 监视文件变动

    publish

  • $ hexo publish [layout] 发表草稿

    server

  • hexo server
  • 启动服务器。默认情况下，访问网址为： http://localhost:4000/。

    选项 描述

  • hexo -p, –port 重设端口
  • hexo -s, –static 只使用静态文件
  • hexo -l, –log 启动日记记录，使用覆盖记录格式

现在我们需要_config.yml文件，来建立关联，命令：

vim _config.yml

翻到最下面，改成我这样子的

deploy:

type: git

repo: https://github.com/leopardpan/leopardpan.github.io.git

branch: master

然后执行命令：

npm install hexo-deployer-git –save

网上会有很多说法，有的type是github, 还有repository最后面的后缀也不一样，是github.com.git，我也踩了很多坑，我现在的版本是hexo: 3.1.1，执行命令hexo -vsersion就出来了,貌似3.0后全部改成我上面这种格式了。

忘了说了，我没用SSH Keys如果你用了SSH Keys的话直接在github里复制SSH的就行了，总共就两种协议，相信你懂的。

然后，执行配置命令：

hexo deploy

然后再浏览器中输入http://leopardpan.github.io/就行了，我的github的账户叫leopardpan,把这个改成你github的账户名就行了

部署步骤

每次部署的步骤，可按以下三步来进行。

hexo clean

hexo generate

hexo deploy

一些常用命令：

hexo new”postName” #新建文章

hexo new page”pageName” #新建页面

hexo generate #生成静态页面至public目录

hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）

hexo deploy #将.deploy目录部署到GitHub

hexo help # 查看帮助

hexo version #查看Hexo的版本

URL

If your site is put in a subdirectory, set url as ‘http://yoursite.com/child‘ and root as ‘/child/‘

title: tags

title: categories
